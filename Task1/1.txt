================================================================
TASK 1 — Complete ML System with Interactive GUI
================================================================

--------------------------
PROBLEM STATEMENT
--------------------------
Banks and financial institutions receive thousands of loan applications every
day. Manually reviewing each application is slow, costly, and can sometimes
be unfair because human reviewers may be inconsistent. The problem is:

    "Can we build a smart system that automatically predicts whether a
     loan application should be APPROVED or REJECTED based on the
     applicant's details?"

This is a classic Binary Classification problem in Machine Learning, where
the answer is either YES (Approved) or NO (Rejected).


--------------------------
OBJECTIVE
--------------------------
The main goals of this task are:

1. Train a machine learning model on real loan application data so it can
   learn patterns that separate approved applications from rejected ones.

2. Compare multiple ML algorithms and pick the one that performs the best.

3. Build an easy-to-use web dashboard (GUI) so that anyone — even someone
   with no coding knowledge — can enter applicant details and instantly
   see a prediction with the approval probability.

In short: Build an end-to-end ML system from raw data → trained model → live
interactive web app.


--------------------------
DATASET DESCRIPTION
--------------------------
Dataset Used : Loan Prediction Dataset (loan_prediction.csv)
Total Records: ~614 rows (loan applications)
Target Column: Loan_Status  (Y = Approved, N = Rejected)

The dataset contains the following features about each applicant:

  NUMERICAL FEATURES (numbers):
  ┌──────────────────────────┬────────────────────────────────────────────┐
  │ Feature                  │ What It Means                              │
  ├──────────────────────────┼────────────────────────────────────────────┤
  │ ApplicantIncome          │ Monthly income of the main applicant       │
  │ CoapplicantIncome        │ Monthly income of the co-applicant         │
  │ LoanAmount               │ Loan amount requested (in thousands)       │
  │ Loan_Amount_Term         │ Repayment period in months (e.g. 360 = 30y)│
  │ Credit_History           │ 1 = good credit history, 0 = bad history   │
  └──────────────────────────┴────────────────────────────────────────────┘

  CATEGORICAL FEATURES (groups/labels):
  ┌──────────────────────────┬────────────────────────────────────────────┐
  │ Feature                  │ What It Means                              │
  ├──────────────────────────┼────────────────────────────────────────────┤
  │ Gender                   │ Male / Female                              │
  │ Married                  │ Yes / No                                   │
  │ Dependents               │ Number of dependents (0, 1, 2, 3+)        │
  │ Education                │ Graduate / Not Graduate                    │
  │ Self_Employed            │ Yes / No                                   │
  │ Property_Area            │ Urban / Semiurban / Rural                  │
  └──────────────────────────┴────────────────────────────────────────────┘

The dataset also had some missing values (blank cells) in several columns,
which we handled as part of data preprocessing.


--------------------------
MY APPROACH
--------------------------
Here is the step-by-step approach followed to complete this task:

STEP 1 — Data Cleaning & Preprocessing
  • Removed the Loan_ID column since it is just a unique identifier and
    doesn't help the model learn anything useful.
  • Dropped rows where the target label (Loan_Status) was missing, because
    we can't train on unlabeled data.
  • Handled missing values:
      - For number columns  → filled with the MEDIAN value of that column.
      - For text columns    → filled with the MOST FREQUENT value (mode).
  • Converted text labels (Y/N) into numbers (1/0) using Label Encoding
    so that ML algorithms can process them.
  • Applied One-Hot Encoding on categorical columns (like Gender, Education)
    to convert them into separate 0/1 columns.
  • Scaled numerical features using StandardScaler so all numbers are on
    the same scale (prevents one large-valued column from dominating).

  All preprocessing steps were packed into a single sklearn Pipeline so
  they can be applied consistently during both training and prediction.

STEP 2 — Model Training
  Three different classification models were trained and compared:

  1. Logistic Regression  — A simple and interpretable linear model.
  2. Random Forest        — An ensemble of many decision trees.
  3. XGBoost              — A powerful gradient boosting algorithm.

  The dataset was split 80% for training and 20% for testing so the model
  is evaluated on data it has never seen before.

STEP 3 — Model Selection
  Each model's accuracy on the test set was measured. The best performing
  model was automatically saved along with the preprocessor into a file
  called best_model.pkl using the joblib library.

STEP 4 — Building the GUI (Web Dashboard)
  A user-friendly web application was built using Streamlit with:
    • A sidebar form where users enter applicant details (income, loan
      amount, credit history, etc.)
    • A Prediction tab that shows APPROVED or REJECTED along with the
      probability score.
    • A Dataset Preview tab so users can explore the raw data.
    • A Dashboard Info tab showing model accuracy comparisons and feature
      importance charts.


--------------------------
FINAL CONCLUSION WITH INSIGHTS
--------------------------

RESULTS SUMMARY:
  Model               | Test Accuracy
  ------------------- | -------------
  Logistic Regression | ~78.86%  ← WINNER (Best Model)
  Random Forest       | ~75–77%
  XGBoost             | ~74–77%

KEY INSIGHTS:

1. Logistic Regression Won Despite Being Simple
   You might expect a complex model like XGBoost to win, but Logistic
   Regression performed the best here. This tells us that the relationship
   between the features and loan approval is largely LINEAR — simple
   patterns are enough to make good predictions on this dataset.

2. Credit History is the Most Powerful Feature
   The single biggest factor in predicting loan approval is whether the
   applicant has a good credit history (Credit_History = 1). Applicants
   with good credit history had a much higher approval rate. This matches
   real-world banking behavior.

3. Income Alone Does NOT Guarantee Approval
   High income doesn't always lead to approval — the loan amount, repayment
   term, and credit history together determine the outcome. So the model
   learned to look at the combination of features, not just income.

4. Missing Data Was Well Handled
   About 8–13% of values were missing in some columns (LoanAmount,
   Loan_Amount_Term, Credit_History, Gender, etc.). The imputation strategy
   (median + mode) worked well and did not hurt model performance.

5. The GUI Makes It Accessible to Everyone
   By wrapping the trained model inside a Streamlit web dashboard, the
   system can now be used by bank employees or analysts who have no coding
   knowledge. They simply fill in a form and get an instant answer.

OVERALL:
  This task successfully demonstrated a complete, real-world ML pipeline —
  from raw messy data to a production-ready interactive prediction system.
  The project covers all core data science skills: data cleaning, feature
  engineering, model comparison, and deployment via a GUI.
