================================================================
TASK 2 — Netflix Movie Recommendation System
================================================================

--------------------------
PROBLEM STATEMENT
--------------------------
With thousands of movies and TV shows available on Netflix, it is nearly
impossible for a user to manually browse through everything and find content
they will actually enjoy. People often waste time scrolling, not knowing
what to watch next.

The problem is:

    "How can we automatically suggest movies or TV shows to a user based
     on something they already like — without needing user ratings or
     watch history?"

This is a recommendation problem, and we solve it using a technique called
Content-Based Filtering — which means we recommend new titles based on
the CONTENT (description, genre, cast, director) of a title the user
already likes, rather than relying on what other users watched.


--------------------------
OBJECTIVE
--------------------------
The main goals of this task are:

1. Build a recommendation engine that takes a movie or TV show as input
   and returns the Top 5 most similar titles from the Netflix dataset.

2. Use Natural Language Processing (NLP) techniques to understand and
   compare the content of each title (description, genres, cast, etc.)

3. Wrap the entire system inside a clean, interactive web application
   (GUI) that anyone can use easily — just pick a title and click a button.

In simple words: "Tell us what you like, and we'll find 5 more things
you'll probably love — automatically."


--------------------------
DATASET DESCRIPTION
--------------------------
Dataset Used : Netflix Titles Dataset (netflix_titles.csv)
Total Records: ~8,800+ movies and TV shows available on Netflix
Source        : Public Netflix catalog data

The dataset contains the following columns (information about each title):

  ┌──────────────────┬────────────────────────────────────────────────────┐
  │ Column           │ What It Means                                      │
  ├──────────────────┼────────────────────────────────────────────────────┤
  │ show_id          │ Unique identifier for each title                   │
  │ type             │ Whether it is a "Movie" or "TV Show"               │
  │ title            │ The name of the movie or TV show                   │
  │ director         │ Who directed it                                    │
  │ cast             │ The actors/actresses in it                         │
  │ country          │ Country where it was produced                      │
  │ date_added       │ When it was added to Netflix                       │
  │ release_year     │ The original release year                          │
  │ rating           │ Age rating (e.g. PG-13, TV-MA, etc.)              │
  │ duration         │ Length in minutes (movies) or seasons (TV shows)  │
  │ listed_in        │ Genre categories (e.g. Drama, Comedy, Thriller)   │
  │ description      │ A short plot summary of the title                 │
  └──────────────────┴────────────────────────────────────────────────────┘

The four columns most important for our recommendation system are:
  → director, cast, listed_in (genres), and description.

These four together describe WHAT the content is about and WHO is in it,
which is exactly what we need to find similar titles.

Note: Several columns had missing values (blank entries), especially
director and cast — we handled these before building the model.


--------------------------
MY APPROACH
--------------------------
Here is the step-by-step approach followed to complete this task:

STEP 1 — Data Loading & Cleaning
  • Loaded the Netflix dataset (8,800+ rows) using pandas.
  • Filled in missing values in director, cast, listed_in, and description
    columns with empty strings so they don't cause errors.

STEP 2 — Feature Engineering (Creating a "Combined Text" Column)
  • Instead of analyzing each column separately, we merged the four key
    columns (director + cast + genres + description) into a single text
    column called "combined_features".
  • Example: For the movie "Inception", the combined_features would be
    something like:
        "christopher nolan leonardo dicaprio action adventure thriller
         a thief who steals corporate secrets through dream sharing..."
  • This single text blob represents the complete identity of each title
    and is what we compare across all titles.

STEP 3 — TF-IDF Vectorization (Converting Text into Numbers)
  • Machine learning models work with numbers, not words.
  • We used a technique called TF-IDF (Term Frequency - Inverse Document
    Frequency) to convert each title's combined_features text into a
    vector (a list of numbers).
  • TF-IDF gives higher weight to words that are UNIQUE to a title and
    lower weight to very common words (called stop words like "the", "is",
    "and") — which are ignored automatically.
  • This gives us a mathematical fingerprint for every title.

STEP 4 — Cosine Similarity (Measuring How Similar Two Titles Are)
  • Once we have number vectors for all titles, we measure how similar
    any two titles are using Cosine Similarity.
  • Cosine Similarity gives a score between 0 and 1:
        → 1.0 = Identical / Perfect match
        → 0.0 = Completely different, nothing in common
  • We computed the similarity score between every pair of titles in
    the dataset. This creates an 8,800 × 8,800 similarity matrix.

STEP 5 — Recommendation Logic
  • When a user picks a title (e.g., "Stranger Things"):
      1. Find that title's row in the similarity matrix.
      2. Sort all other titles by their similarity score (highest first).
      3. Return the Top 5 most similar titles (skipping the title itself).

STEP 6 — Building the GUI (Streamlit Web App)
  • Built an interactive, Netflix-themed web dashboard using Streamlit.
  • Users can search and select any title from a dropdown list.
  • After clicking "Get Recommendations", the app displays 5 cards with:
      - Title name
      - Match percentage (e.g. 🔥 87.3% Match)
      - Type (Movie / TV Show), Release Year, and Genres
      - A short description of the recommended title
  • The UI uses a dark theme with Netflix red (#E50914) and hover effects
    for a premium feel.


--------------------------
FINAL CONCLUSION WITH INSIGHTS
--------------------------

HOW WELL DID IT WORK?
  The recommendation system runs entirely on the CONTENT of titles (no user
  data needed), and it produces logically sensible recommendations:
    • Searching for a crime thriller suggests other crime thrillers.
    • Searching for a stand-up comedy special suggests similar specials.
    • Searching for a title by a specific director may surface other movies
      by the same director, since director name is part of the features.

KEY INSIGHTS:

1. Content-Based Filtering Works Without User Data
   Unlike platforms that need your watch history or ratings, this system
   works purely on WHAT the content is about. This is great for new users
   or cold-start scenarios where no personal data is available yet.

2. TF-IDF + Cosine Similarity is a Simple but Effective Combination
   Despite being a relatively old technique, TF-IDF + Cosine Similarity
   gives surprisingly relevant results for text-based recommendations.
   It is fast, easy to understand, and requires no model training time.

3. Description + Genre + Cast = Strong Content Signal
   Combining description, genres, director, and cast into one text field
   creates a rich representation of each title. Genres alone would miss
   subtle differences; description alone would miss cast patterns. Together,
   they work much better.

4. The Similarity Matrix is Resource-Intensive
   Computing a full 8,800 × 8,800 cosine similarity matrix takes some
   memory and time on first load. We solved this by using Streamlit's
   @st.cache_resource decorator, which caches the result so it only
   computes once per session rather than on every interaction.

5. Limitation — It Doesn't Learn From User Feedback
   Since this is content-based only, it doesn't improve over time or
   adapt to individual taste. It will always recommend the same 5 titles
   for the same input. A future improvement would be to add Collaborative
   Filtering (based on what similar users watched) to personalize results.

6. The GUI Makes it Instantly Usable
   The Streamlit web app removes the technical barrier completely. A user
   doesn't need to know anything about ML or Python — they just open the
   app, pick a title, and get recommendations in seconds.

OVERALL:
  This task successfully demonstrated how NLP and similarity algorithms
  can power a recommendation engine without any machine learning model
  training. The system is fast, interpretable, and produces meaningful
  results from a large real-world dataset of 8,800+ Netflix titles.
  The interactive GUI brings the technology to life in a simple, elegant
  way that anyone can use.
